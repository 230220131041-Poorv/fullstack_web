CSS Grid — Introduction
Two-dimensional layout — the most powerful CSS layout system

What is CSS Grid?

CSS Grid Layout is a two-dimensional layout system — it controls both rows AND columns at the same time. This is what makes it different from Flexbox, which is only one-dimensional (either row OR column at a time).

With Grid, you divide a container into a grid of rows and columns, then place items precisely anywhere in that grid — including spanning multiple rows or columns. It is ideal for page-level layouts — the big structural skeleton of a webpage.

Key mental model: Imagine a spreadsheet. CSS Grid turns an HTML element into a spreadsheet-like structure where you control the rows, columns, and where each child element sits — including merging cells (spanning). You define the grid on the parent; children place themselves in it.

Grid Terminology You Must Know

Grid Container — the element with display: grid. Parent of all grid items.
Grid Item — direct children of the grid container.
Grid Line — the dividing lines making up the grid structure (numbered 1 to n+1).
Grid Track — the space between two adjacent grid lines (a row or column).
Grid Cell — single intersection of a row and column — the smallest unit.
Grid Area — any rectangular space bounded by 4 grid lines (can span multiple cells).

Grid Container Properties
Defining rows, columns, gaps, and alignment on the parent
CSS — Grid Container Properties (Complete) 
.grid-container {
  display: grid;         /* or inline-grid */

  /* ── DEFINING COLUMNS ── */
  grid-template-columns: 200px 400px 200px;    /* 3 fixed columns */
  grid-template-columns: 1fr 2fr 1fr;           /* fr = fraction of remaining space */
  grid-template-columns: repeat(3, 1fr);       /* 3 equal columns */
  grid-template-columns: repeat(4, 250px);     /* 4 columns of 250px each */
  grid-template-columns: 260px repeat(3, 1fr); /* sidebar + 3 equal */
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
                                                 /* Responsive! Fill as many as fit, min 280px */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
                                                 /* Like auto-fill but collapses empty tracks */

  /* ── DEFINING ROWS ── */
  grid-template-rows: 60px 1fr 80px;  /* header height, main flex, footer height */
  grid-template-rows: repeat(3, 200px);
  grid-auto-rows: minmax(120px, auto); /* implicitly created rows: min 120px, grow as needed */

  /* ── SHORTHAND: rows / columns ── */
  grid-template: 60px 1fr 60px / 240px 1fr; /* rows / columns */

  /* ── GAPS ── */
  gap: 24px;              /* row-gap AND column-gap */
  gap: 16px 24px;        /* row-gap column-gap */
  row-gap: 16px;
  column-gap: 24px;

  /* ── ALIGNMENT OF ALL ITEMS (inside their cells) ── */
  justify-items: start | end | center | stretch; /* horizontal in cell */
  align-items:   start | end | center | stretch; /* vertical in cell */
  place-items: center;   /* shorthand: align-items justify-items */

  /* ── ALIGNMENT OF GRID ITSELF (when grid smaller than container) ── */
  justify-content: start | end | center | space-between | space-evenly;
  align-content:   start | end | center | space-between;
  place-content:   center;  /* shorthand: align-content justify-content */

  /* ── NAMED TEMPLATE AREAS (visual layout definition!) ── */
  grid-template-areas:
    "header  header  header"
    "sidebar main    main  "
    "footer  footer  footer";
  /* Each string = one row. Each word = which area that cell belongs to */
  /* Use '.' for an empty cell: "sidebar . main" */
}

The fr Unit — The Most Important Grid Unit

The fr (fraction) unit represents a fraction of the available space in the grid container. It's calculated after fixed-size columns are accounted for.

Example: grid-template-columns: 200px 1fr 2fr — first column is fixed 200px. Remaining space is split: second column gets 1 part, third gets 2 parts. If container is 800px, remaining is 600px → second gets 200px, third gets 400px.

minmax() — Responsive Without Media Queries

minmax(min, max) defines a size range. Tracks can be at least min and at most max. Combined with auto-fill or auto-fit, it creates fully responsive grids without a single media query.

CSS — Responsive Grid without Media Queries 
/* This single line creates a fully responsive card grid! */
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 24px;
}
/*
  On 1200px screen: fits 4 cards (4 × 280px = 1120px fits)
  On 900px screen:  fits 3 cards
  On 600px screen:  fits 2 cards
  On 320px screen:  fits 1 card
  No media queries needed! 🎉
*/

/* auto-fill vs auto-fit:
  auto-fill: keeps empty track space (tracks still exist)
  auto-fit:  collapses empty tracks → items stretch to fill
  For most cases, use auto-fit for card grids
*/
.product-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
  gap: 20px;
}

Grid Item Placement
Precisely positioning and spanning items in the grid
CSS — Grid Item Properties 
/* ── GRID LINES: numbered from 1 (positive) or -1 (from end) ── */
.item-a {
  /* Using line numbers */
  grid-column-start: 1;   /* start at column line 1 */
  grid-column-end: 3;     /* end at column line 3 (spans 2 columns) */
  grid-row-start: 1;
  grid-row-end: 2;

  /* Shorthand: start / end */
  grid-column: 1 / 3;   /* start at line 1, end at line 3 */
  grid-row: 1 / 2;

  /* Using span keyword */
  grid-column: 1 / span 2;  /* start line 1, span 2 columns */
  grid-column: span 2;       /* auto-place but span 2 columns */
  grid-row: span 2;           /* span 2 rows */

  /* Using -1 to mean "to the very end" */
  grid-column: 1 / -1;       /* full width — start to end */

  /* Shorthand: grid-area: row-start / col-start / row-end / col-end */
  grid-area: 1 / 1 / 2 / 3;  /* row 1, col 1 to row 2, col 3 */

  /* Per-item alignment (override container's justify/align-items) */
  justify-self: start | end | center | stretch;
  align-self:   start | end | center | stretch;
  place-self: center;  /* shorthand */
}

/* ── USING NAMED AREAS ── */
.layout {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-rows: 64px 1fr 48px;
  grid-template-areas:
    "header header"

    "footer footer";
  min-height: 100vh;
}

.site-header  { grid-area: header; }   /* placed in the "header" area */
.site-sidebar { grid-area: sidebar; }

CSS Grid vs Flexbox
When to use which — the most common interview question

The Golden Rule

This question is asked in almost every CSS interview. The answer is not "one is better" — they are complementary tools designed for different jobs. A real-world page uses both at different levels of the hierarchy.

Use CSS Grid when…

✦ You have a two-dimensional layout — rows AND columns matter simultaneously.

✦ You're doing page-level layout — headers, sidebars, main content, footers.

✦ You need items to align across both axes at the same time.

✦ You want to define the structure first and place items into it (content-agnostic).

✦ Example: A dashboard grid, a magazine layout, a photo gallery.

Use Flexbox when…

✦ You have a one-dimensional layout — a row of items OR a column of items.

✦ You're doing component-level layout — inside a card, a navbar, a button group.

✦ You want items to flex and grow based on their content.

✦ You need items to wrap dynamically with flexible sizing.

✦ Example: A navbar, a button group, a form row, a list of badges.

Real-world usage: Use Grid for the overall page skeleton (header/sidebar/main/footer). Use Flexbox inside each section (navbar items, card content layout, form controls). They nest perfectly and complement each other.
🏢 Real World

How Ahmedabad Companies Use Grid & Flexbox Together

In a typical SaaS dashboard: CSS Grid defines the outer shell — sidebar is 260px, content area takes 1fr, header is 60px tall. Inside the content area, CSS Grid again creates the card grid with auto-fit. Inside each card, Flexbox aligns the card header, body, and footer. Inside the navbar (which is itself a Grid item), Flexbox positions the logo, links, and user avatar.

CSS Transitions
Smooth, automatic animation between two states

What is a Transition?

A CSS transition smoothly animates a property from its current value to a new value when a change is triggered — such as a hover, focus, or class toggle. You define what properties to animate, how long, and how they ease. The browser handles all the in-between frames automatically.

Think of it as: "When this property changes, animate the change smoothly over X seconds instead of instantly."

CSS — Transitions Complete 
.btn {
  background: #4facfe;
  color: white;
  padding: 12px 24px;
  border-radius: 8px;

  /* ── TRANSITION SYNTAX ── */
  /* transition: property  duration  timing-function  delay; */
  transition: background 0.3s ease;

  /* Multiple properties */
  transition:
    background   0.3s ease,
    transform    0.2s ease,
    box-shadow   0.3s ease;

  /* Transition ALL animatable properties (use carefully) */
  transition: all 0.3s ease;
}

.btn:hover {
  background: #3d8fd4;
  transform: translateY(-2px);
  box-shadow: 0 8px 20px rgba(79,172,254,0.4);
}

/* ── TIMING FUNCTIONS (easing) ── */
transition-timing-function: ease;          /* slow → fast → slow (default) */
transition-timing-function: ease-in;       /* slow start, fast end */
transition-timing-function: ease-out;      /* fast start, slow end */
transition-timing-function: ease-in-out;   /* slow start, slow end */
transition-timing-function: linear;        /* constant speed */
transition-timing-function: cubic-bezier(0.25, 0.46, 0.45, 0.94);
                                              /* Custom bezier curve */
transition-timing-function: steps(4);      /* Jumpy (sprite animation) */

/* ── INDIVIDUAL PROPERTIES ── */
transition-property: background, transform;
transition-duration: 0.3s;
transition-timing-function: ease;
transition-delay: 0.1s;  /* wait 100ms before starting */

/* ── COMMON ANIMATABLE PROPERTIES ── */
/* opacity, transform, color, background-color, width, height,
   border-radius, box-shadow, font-size, letter-spacing, margin, padding */
/* NOT animatable: display, visibility (use opacity instead) */

/* ── PERFORMANCE TIP: Only transition opacity and transform ── */
/* These run on the GPU compositor (no layout recalculation) */
/* Animating width/height/margin causes layout reflow = slower */
.performant {
  transition: opacity 0.3s ease, transform 0.3s ease;
}

/* ── WILL-CHANGE: hint to browser (use sparingly) ── */
.heavy-animation { will-change: transform; }

CSS Transforms
Move, scale, rotate, and skew elements — without affecting layout

What is a CSS Transform?

CSS transforms visually move, resize, rotate, or distort an element — without affecting the document layout. Other elements don't move to compensate. This is why transforms are the preferred tool for animation — they're GPU-accelerated and don't cause layout recalculations (reflow).

CSS — All Transform Functions 
/* ── TRANSLATE (move) ── */
transform: translateX(50px);           /* move right 50px */
transform: translateY(-20px);          /* move up 20px */
transform: translate(50px, -20px);    /* move right 50px, up 20px */
transform: translate(-50%, -50%);    /* centre trick (used with position:absolute) */

/* ── SCALE (resize) ── */
transform: scale(1.2);               /* 120% of original size */
transform: scale(1.2, 0.8);          /* 120% wide, 80% tall */
transform: scaleX(2);               /* double width */
transform: scaleY(0.5);             /* half height */

/* ── ROTATE ── */
transform: rotate(45deg);            /* 45 degrees clockwise */
transform: rotate(-90deg);           /* 90 degrees counter-clockwise */
transform: rotate(0.5turn);          /* half turn = 180deg */

/* ── SKEW ── */
transform: skewX(20deg);             /* slant horizontally */
transform: skewY(10deg);             /* slant vertically */
transform: skew(20deg, 10deg);      /* both */

/* ── COMBINING (apply right to left!) ── */
transform: translateY(-4px) scale(1.02);  /* lift up + enlarge */
transform: rotate(45deg) scale(1.5);      /* rotate then scale */

/* ── 3D TRANSFORMS ── */
transform: rotateX(60deg);           /* tilt backward/forward */
transform: rotateY(45deg);           /* spin left/right */
transform: perspective(800px) rotateY(30deg);

/* ── TRANSFORM ORIGIN: pivot point for rotation/scale ── */
transform-origin: center;      /* default: 50% 50% */
transform-origin: top left;   /* rotate from top-left corner */
transform-origin: 0 0;         /* same as top left */
transform-origin: 100% 50%;   /* right-middle edge */

/* ── REAL PATTERN: Card hover lift ── */
.card {
  transition: transform 0.3s ease, box-shadow 0.3s ease;
}
.card:hover {
  transform: translateY(-6px);
  box-shadow: 0 20px 40px rgba(0,0,0,0.15);
}

/* ── REAL PATTERN: Rotating hamburger to X icon ── */
.hamburger-line      { transition: transform 0.3s ease, opacity 0.3s; }
.open .line-1        { transform: translateY(7px) rotate(45deg); }
.open .line-2        { opacity: 0; }
.open .line-3        { transform: translateY(-7px) rotate(-45deg); }

CSS Animations — @keyframes
Multi-step, looping, and complex animations with full control

Transition vs Animation

Transitions need an external trigger (hover, focus, class change) and animate between exactly 2 states (A → B).
Animations can start automatically, loop indefinitely, have multiple keyframe steps (A → B → C → D), run in reverse, and have precise per-step control. You define them with @keyframes.

CSS — @keyframes Animations Complete 
/* ── STEP 1: Define the animation with @keyframes ── */
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

/* Multiple stops with percentages */
@keyframes bounceIn {
  0%   { transform: scale(0); opacity: 0; }
  60%  { transform: scale(1.1); opacity: 1; }
  80%  { transform: scale(0.95); }
  100% { transform: scale(1); }
}

@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

@keyframes pulse {
  0%, 100% { transform: scale(1); opacity: 1; }
  50%      { transform: scale(1.05); opacity: 0.8; }
}

/* ── STEP 2: Apply the animation ── */
.element {
  /* animation: name  duration  timing  delay  iteration  direction  fill-mode */
  animation: fadeIn 0.6s ease 0s 1 normal both;

  /* Common shorthand */
  animation: fadeIn 0.6s ease;
  animation: spin 2s linear infinite;  /* spinner */
  animation: pulse 1.5s ease-in-out infinite;
  animation: bounceIn 0.8s cubic-bezier(0.36, 0.07, 0.19, 0.97) both;
}

/* ── INDIVIDUAL PROPERTIES ── */
animation-name:            fadeIn;
animation-duration:        0.6s;
animation-timing-function: ease;
animation-delay:           0.2s;
animation-iteration-count: 1 | infinite | 3;
animation-direction:       normal | reverse | alternate | alternate-reverse;
animation-fill-mode:       none | forwards | backwards | both;
  /* forwards: keeps end state after animation finishes */
  /* backwards: applies from-state during delay period */
  /* both: both of the above */
animation-play-state:      running | paused;

/* Pause animation on hover */
.loading:hover { animation-play-state: paused; }

/* Multiple animations on one element */
.element { animation: spin 2s linear infinite, pulse 3s ease infinite; }

CSS — Overflow, Scroll & Clipping 
/* ── OVERFLOW ── */
.box {
  overflow: visible; /* default: content spills out */
  overflow: hidden;  /* clips content, no scrollbar */
  overflow: scroll;  /* always shows scrollbar */
  overflow: auto;    /* scrollbar only when needed (BEST) */
  overflow-x: hidden; overflow-y: auto; /* independent axes */
}

/* ── TEXT TRUNCATION ── */
.truncate {
  white-space: nowrap;    /* prevent wrapping */

}

/* Multi-line truncation */
.clamp-3 {

Placement Interview Q&A
Top CSS Grid & Animation questions from real company interviews
  • Q1. What is the difference between CSS Grid and Flexbox?
    Flexbox is one-dimensional — it arranges items in a single row OR column. Grid is two-dimensional — it controls rows AND columns simultaneously. Use Flexbox for component-level layout (navbars, card internals, button groups) where you're arranging items along one axis. Use Grid for page-level layout (dashboard shells, magazine layouts, card grids) where you need to control both dimensions at once. In real projects, you use both together — Grid for the page skeleton, Flexbox inside each section.
  • Q2. What is the fr unit in CSS Grid?
    The fr (fraction) unit represents a fraction of the available space in the grid container after fixed-size columns/rows are accounted for. For example, grid-template-columns: 1fr 2fr means the available space is divided into 3 parts: first column gets 1 part (~33%), second gets 2 parts (~67%). It's like flex-grow but for grid tracks. It's the key to creating flexible, proportional grids without fixed pixel widths.
  • Q3. How do you make a responsive grid without media queries?
    Using repeat() with auto-fit or auto-fill and minmax(): grid-template-columns: repeat(auto-fit, minmax(280px, 1fr)). This creates as many columns as fit in the container, each at minimum 280px and maximum 1fr. On a 1200px screen it fits 4 cards; on 600px it fits 2; on 320px it shows 1 — all without any media queries. auto-fit collapses empty tracks so items stretch; auto-fill keeps empty track space.
  • Q4. What is the difference between CSS transitions and CSS animations?
    Transitions require a trigger (hover, focus, class toggle) and animate between exactly two states. They're simpler and defined with the transition property. Animations use @keyframes to define multiple steps and can: run automatically on page load, loop infinitely, run in reverse, have precise per-frame control, and be paused or replayed. For a hover effect, use transition. For a loading spinner, page entrance, or looping effect, use animation.
  • Q5. What is grid-template-areas and why is it useful?
    grid-template-areas lets you define a grid layout visually using named regions. You write strings where each word is the area name and each string is a row: "header header" / "sidebar main" / "footer footer". Then assign items to areas with grid-area: header. It makes the layout self-documenting — you can see the page structure at a glance in CSS. Changing the layout for different screen sizes is as simple as rewriting the template string in a media query.
  • Q6. Why should you prefer animating transform and opacity over width or background?
    Transform (translate, scale, rotate) and opacity are handled by the browser's compositor on the GPU — they don't cause layout recalculation or repaint. Animating properties like width, height, margin, top, or left causes the browser to recalculate layout (reflow) on every frame, which is expensive and causes jank on low-powered devices. As a rule: for smooth 60fps animations, use transform and opacity. If you need to show/hide something, use opacity: 0 + pointer-events: none instead of display: none (which can't be transitioned).
  • Q7. What is animation-fill-mode: forwards and when do you use it?
    By default, when a CSS animation ends, the element snaps back to its pre-animation styles. animation-fill-mode: forwards tells the browser to keep the final keyframe styles applied after the animation completes. You use this when you want an entrance animation (like a fade-in or slide-up) to keep its result — so the element stays visible and in its final position after animating in. Without it, elements would flash back to their invisible/off-screen starting state.
  • Q8. What is a skeleton screen and how is it implemented with CSS?
    A skeleton screen is a loading placeholder that mimics the shape of the content that's loading — grey blocks in place of text, images, and cards. It uses a shimmer animation: a linear-gradient from grey to lighter grey to grey, with background-size: 200%, animated with background-position moving from -200% to 200% repeatedly. This gives the illusion of a light sweeping across the placeholder. It's better UX than a spinner because it shows users the expected layout structure while they wait.
