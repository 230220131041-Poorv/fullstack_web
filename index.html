What is HTML?
HyperText Markup Language — the foundation of the web

Definition

HTML stands for HyperText Markup Language. It is the standard language used to create and structure content on the web. HTML is NOT a programming language — it is a markup language that tells the browser how to display content.

Think of HTML as the skeleton of a website. CSS is the skin (styling), and JavaScript is the muscles (behaviour). Without HTML, there's nothing to style or interact with.

🏢 Real-World Analogy: HTML is like the blueprint of a building. It defines — here is a room (div), here is a door (anchor tag), here is a window (img). The interior design (CSS) and electrical wiring (JS) come later.

How the Browser Reads HTML

When you open a website, the browser makes an HTTP request to a server. The server responds with an HTML file. The browser then parses (reads) the HTML top-to-bottom and builds a DOM (Document Object Model) — a tree structure of all the elements. This DOM is what you see rendered on screen.

Key concept: The DOM is not the same as your HTML file. It is the browser's live, in-memory representation of your HTML that JavaScript can read and change.

HTML Element Anatomy

Every HTML element has the same basic anatomy:

HTML — Element Anatomy 
<!-- Basic Element: Opening Tag + Content + Closing Tag -->

<!-- Element with Attributes -->

<input type="text" />

<!-- Nested Elements (Parent → Child) -->
<div>          <!-- Parent -->
  <h1>Title</h1>  <!-- Child -->

</div>

HTML Document Structure
The mandatory skeleton every webpage must have

The Boilerplate

Every valid HTML5 document must start with specific declarations. These are not optional — the browser needs them to correctly render your page. This standard starting template is called the boilerplate.

HTML — Full Boilerplate 
<!DOCTYPE html>                     <!-- Tells browser: this is HTML5 -->
<html lang="en">                   <!-- Root element; lang for accessibility/SEO -->

  <head>                             <!-- Meta info — NOT visible on page -->
    <meta charset="UTF-8" />         <!-- Supports all characters including Hindi/Gujarati -->
    <meta name="viewport"
          content="width=device-width, initial-scale=1.0" />
                                       <!-- Makes page responsive on mobile -->
    <meta name="description"
          content="My website description for Google" />
    <title>My Website</title>          <!-- Shows in browser tab -->
    <link rel="stylesheet" href="style.css" />
  </head>

  <body>                             <!-- Everything visible goes here -->
    <h1>Hello World!</h1>
    <script src="script.js"></script> <!-- JS at end of body for performance -->
  </body>

</html>
<head> tag

Contains metadata — information about the page that the browser and search engines need, but the user never directly sees. Title, CSS links, meta tags, favicon all go here.

<body> tag

Everything the user sees on the page lives here — headings, paragraphs, images, buttons, forms, everything. This is what gets rendered visually in the browser window.

charset="UTF-8"

UTF-8 supports every character in every language — English, Hindi, Gujarati, Chinese, Arabic, emojis — all encoded correctly. Without this, special characters may break.

viewport meta

Critical for mobile responsiveness. Without it, mobile browsers zoom out the desktop version. With it, the browser uses the device's actual pixel width for layout.

Semantic HTML5 Tags
Tags that carry meaning — critical for SEO and accessibility

What is Semantic HTML?

Before HTML5, developers used <div> for everything — navigation, articles, sidebars, footers. The problem: a <div> tells the browser nothing about what the content is.

Semantic tags describe the role of the content. This helps:

  • Search engines (Google) understand your page structure → better SEO ranking
  • Screen readers for visually impaired users navigate correctly → Accessibility
  • Other developers quickly understand the page layout → Maintainability
Rule of thumb: Always ask "does this tag describe what the content IS?" If yes, it's semantic. <article> = self-contained content. <nav> = navigation links. <div> = nothing (generic container).
Tag Purpose When to use
<header> Page or section header Logo, site name, top nav
<nav> Navigation links Menu, breadcrumbs, pagination
<main> Primary page content One per page, the central content
<section> Thematic grouping Features section, About section
<article> Self-contained content Blog post, news article, product card
<aside> Secondary/related content Sidebar, related posts, ads
<footer> Page or section footer Copyright, links, social icons
<figure> Media with caption Image + description, diagram
<time> Date/time representation Published date on articles

Text & Heading Elements
Typography structure — from h1 to span

Headings (h1–h6)

HTML has 6 levels of headings. <h1> is the most important (largest), <h6> is the least. Each page should have exactly one <h1> — it tells Google and screen readers what the page's main topic is. Use h2–h6 as a logical hierarchy.

SEO rule: Never skip heading levels. Don't jump from h1 to h4. Google reads heading hierarchy to understand your content's structure.
HTML — Text Elements 
<!-- Headings -->
<h1>Page Title (only one per page)</h1>
<h2>Major Section</h2>
<h3>Sub-section</h3>
<h4>Sub-sub-section</h4>

<!-- Paragraph -->
<p>This is a paragraph of text. Use for body content.</p>

<!-- Inline text elements -->
<p>
  This is <strong>important text</strong> (bold + semantic weight).
  This is <em>emphasised text</em> (italic + semantic stress).
  This is <b>bold</b> (just visual, no semantic meaning).
  This is <i>italic</i> (just visual, no semantic meaning).
  This is <mark>highlighted</mark>.
  This is <code>inline code</code>.
  This is <del>deleted text</del> and <ins>inserted text</ins>.
  This is X<sup>2</sup> (superscript) and H<sub>2</sub>O (subscript).
</p>

<!-- Blockquote -->
<blockquote cite="https://source.com">
  First, solve the problem. Then, write the code.
</blockquote>

<!-- Preformatted text (preserves spaces and line breaks) -->
<pre>
  function hello() {
    console.log("hi");
  }
</pre>

<!-- Line break and horizontal rule -->
<br />   <!-- Line break -->
<hr />   <!-- Horizontal divider line -->

Links & Images
Connecting pages and displaying media

The Anchor Tag — <a>

Anchor tags create hyperlinks. The href attribute specifies where the link goes. Links can go to another page, a section on the same page (using #id), an email address, or a phone number.

HTML — Links & Images 
<!-- External link (opens in new tab) -->
<a href="https://github.com" target="_blank" rel="noopener noreferrer">
  GitHub
</a>
<!-- rel="noopener noreferrer" is a SECURITY BEST PRACTICE for target="_blank" -->

<!-- Internal link (same website) -->
<a href="/about">About Us</a>

<!-- Jump to section on same page -->
<a href="#contact">Go to Contact Section</a>

<!-- Email and phone links -->
<a href="mailto:poorv@email.com">Email Me</a>
<a href="tel:+919876543210">Call Me</a>

<!-- ──────────────────────────────── -->
<!-- IMAGES -->

<!-- Basic image -->
<img
  src="./images/profile.jpg"
  alt="Poorv's profile photo"    <!-- REQUIRED: for screen readers + broken img -->
  width="300"
  height="300"
  loading="lazy"               <!-- Performance: loads only when visible -->
/>

<!-- Image with caption (semantic) -->
<figure>
  <img src="architecture.png" alt="System Architecture Diagram" />
  <figcaption>Fig 1: Three-tier web architecture</figcaption>
</figure>

<!-- Responsive image (shows different sizes on different screens) -->
<img
  src="banner-large.jpg"
  srcset="banner-small.jpg 480w, banner-medium.jpg 800w, banner-large.jpg 1200w"
  sizes="(max-width: 600px) 480px, (max-width: 900px) 800px, 1200px"
  alt="Banner"
/>
Interview Tip: Always add alt text to images. It's required for accessibility (WCAG compliance) and helps Google understand image content. Companies that care about quality will ask about this.

  <caption>Employee List</caption>

      <th scope="col">Name</th>    <!-- Table Header cell -->
      <th scope="col">Role</th>

HTML Forms & Inputs
Collecting user data — the most important HTML topic for placements

Why Forms Matter

Forms are the primary way users interact with web applications — login, signup, search, checkout, contact. Almost every company product uses forms. Interviewers specifically ask about form attributes because it shows whether you understand HTTP and server communication.

method="GET" — Data sent in URL. Use for search queries (not sensitive). Bookmarkable.
method="POST" — Data sent in request body. Use for login, signup, file upload (sensitive data).
HTML — Complete Registration Form 
<form action="/api/register" method="POST" enctype="multipart/form-data">

  <!-- Text input -->
  <label for="name">Full Name</label>
  <input
    type="text"
    id="name"
    name="fullName"
    placeholder="Enter your full name"
    required
    minlength="2"
    maxlength="50"
    autocomplete="name"
  />

  <!-- Email input -->
  <label for="email">Email</label>
  <input type="email" id="email" name="email" required />

  <!-- Password input -->
  <label for="password">Password</label>
  <input type="password" id="password" name="password"
         minlength="8" required />

  <!-- Number input -->
  <input type="number" name="age" min="18" max="100" />

  <!-- Phone input -->
  <input type="tel" name="phone" pattern="[0-9]{10}" />

  <!-- Date input -->
  <input type="date" name="dob" min="1990-01-01" />

  <!-- File upload -->
  <input type="file" name="resume" accept=".pdf,.doc" />

  <!-- Textarea (multiline text) -->
  <label for="bio">About You</label>
  <textarea id="bio" name="bio" rows="4" cols="50"></textarea>

  <!-- Dropdown select -->
  <label for="city">City</label>
  <select id="city" name="city">
    <option value="">-- Select City --</option>
    <option value="ahmedabad">Ahmedabad</option>
    <option value="surat">Surat</option>
    <option value="vadodara">Vadodara</option>
    <option value="junagadh" selected>Junagadh</option>
  </select>

  <!-- Radio buttons (choose one) -->
  <fieldset>
    <legend>Gender</legend>
    <input type="radio" id="male" name="gender" value="male"/>
    <label for="male">Male</label>
    <input type="radio" id="female" name="gender" value="female"/>
    <label for="female">Female</label>
  </fieldset>

  <!-- Checkboxes (choose multiple) -->
  <fieldset>
    <legend>Skills</legend>
    <input type="checkbox" id="html" name="skills" value="html" checked/>
    <label for="html">HTML</label>
    <input type="checkbox" id="css" name="skills" value="css"/>
    <label for="css">CSS</label>
    <input type="checkbox" id="js" name="skills" value="javascript"/>
    <label for="js">JavaScript</label>
  </fieldset>

  <!-- Submit button -->
  <button type="submit">Register</button>
  <button type="reset">Clear Form</button>

</form>

Placement Interview Q&A
Top HTML questions asked in Ahmedabad company interviews
  • Q1. What is the difference between <div> and semantic tags like <section> or <article>?
    A <div> is a generic, non-semantic container that carries no meaning. Semantic tags like <section>, <article>, <nav> describe the role of their content. Search engines and screen readers can understand semantic tags, which improves SEO and accessibility. In modern development, you should always prefer semantic tags over bare <div>s when a tag's purpose matches your content.
  • Q2. What is the difference between GET and POST in forms?
    GET sends form data as URL query parameters (visible in the address bar). It's used for non-sensitive data like search queries and is bookmarkable. POST sends data in the request body — hidden from the URL. Use POST for login, signup, file uploads, or any sensitive operation. GET has URL length limits; POST has no practical size limit.
  • Q3. What is the purpose of the alt attribute on images?
    The alt (alternative text) attribute serves two purposes: (1) It's read by screen readers for visually impaired users, making your site accessible. (2) It's displayed as fallback text when the image fails to load. It also helps search engines index your images for image search. Always write descriptive alt text — not just "image" or "photo".
  • Q4. What is the DOM?
    DOM stands for Document Object Model. When the browser reads your HTML file, it parses it and creates a tree-like in-memory structure called the DOM. Each HTML element becomes a "node" in this tree. JavaScript interacts with the DOM (not the HTML file directly) to read or change page content dynamically — for example, document.getElementById(), document.querySelector().
  • Q5. Why do we put <script> at the bottom of <body> instead of inside <head>?
    HTML is parsed top-to-bottom. If a <script> is in <head>, the browser pauses HTML parsing to download and execute it — the user sees a blank page. Putting scripts at the bottom of <body> means all the HTML is parsed and displayed first, then JS runs. Alternatives are the defer attribute (downloads in background, runs after DOM ready) or async (runs as soon as downloaded).
  • Q6. What is the purpose of <!DOCTYPE html>?
    DOCTYPE is not an HTML tag — it's an instruction to the browser. It tells the browser which version of HTML the page is written in. <!DOCTYPE html> is the HTML5 doctype. Without it, some browsers enter "quirks mode" and apply old rendering behaviour from the 1990s, which can break your layout unexpectedly.
  • Q7. What are data attributes and when would you use them?
    Data attributes (data-*) let you store custom data on HTML elements that JavaScript can later read. For example: <div data-product-id="42" data-price="999">. In JavaScript: element.dataset.productId gives "42". This is useful when you need to pass dynamic backend data (like IDs or config values) into the HTML that your JS frontend can use without making an extra API call.
  • Q8. What is the difference between <strong> and <b>, and <em> vs <i>?
    <strong> and <em> are semantic — they carry meaning. <strong> means the text has strong importance (screen readers may stress it). <em> means emphasis (vocal stress). Both look like bold and italic by default. <b> and <i> are purely visual — they make text bold or italic with no semantic meaning. Use <strong>/<em> when the meaning matters, <b>/<i> for stylistic purposes only.
