Arrays — Deep Dive
Ordered lists of values — the most used data structure in JS

What is an Array?

An array is an ordered, zero-indexed collection of values. In JavaScript, arrays are actually special objects — that's why typeof [] returns "object". Arrays can hold mixed types (numbers, strings, objects, even other arrays) since JS is dynamically typed.

JavaScript — Array Basics
// ── CREATING ARRAYS ──
const fruits = ['apple', 'banana', 'mango'];
const mixed  = [1, 'two', true, { id: 4 }, [5, 6]];
const empty  = [];
const arr2   = new Array(5);       // creates array with 5 empty slots
const filled = Array.from({length: 5}, (_, i) => i); // [0,1,2,3,4]

// ── ACCESSING & MODIFYING ──
fruits[0]                  // 'apple' (zero-indexed)
fruits[fruits.length - 1]  // 'mango' (last item)
fruits.at(-1)             // 'mango' (modern way to get last item)
fruits[0] = 'orange';        // modify in place
fruits.length               // 3

// ── ADDING / REMOVING (mutating methods) ──
fruits.push('grape');        // add to END → returns new length
fruits.pop();                // remove from END → returns removed item
fruits.unshift('kiwi');      // add to START → returns new length
fruits.shift();              // remove from START → returns removed item

// splice(start, deleteCount, ...itemsToAdd) — swiss-army knife
fruits.splice(1, 1);          // remove 1 item at index 1
fruits.splice(1, 0, 'pear');  // insert 'pear' at index 1 (delete 0)
fruits.splice(1, 2, 'fig');   // replace 2 items starting at index 1

// ── NON-MUTATING (return NEW array, original unchanged) ──
fruits.slice(1, 3);            // new array from index 1 to 2 (end exclusive)
fruits.slice(-2);              // last 2 items
fruits.concat(['x', 'y']);     // merge arrays (returns new)
[...fruits, 'x'];           // merge using spread (modern way)
fruits.toReversed();         // reversed copy (ES2023) — non-mutating
fruits.reverse();            // ⚠️ MUTATES original array, reverses in place

// ── SEARCHING ──
fruits.indexOf('mango');     // index, or -1 if not found
fruits.includes('mango');   // true/false
fruits.find(f => f.startsWith('m')); // first matching ELEMENT or undefined
fruits.findIndex(f => f === 'mango'); // first matching INDEX or -1
fruits.findLast(f => f.length > 5);   // last matching element

// ── CHECKING TYPE ──
Array.isArray(fruits);    // true — reliable array check
Array.isArray('hello');  // false

// ── SORTING (⚠️ MUTATES original array!) ──
const nums = [5, 2, 8, 1];
nums.sort();                  // [1, 2, 5, 8] — default: lexicographic (string) order!
[10, 2, 1].sort();         // [1, 10, 2] ← BUG: sorts as strings by default
nums.sort((a, b) => a - b); // ascending numeric: [1, 2, 5, 8]
nums.sort((a, b) => b - a); // descending numeric: [8, 5, 2, 1]

// Sort objects by property
const users = [{name:'Bob',age:25}, {name:'Amy',age:30}];
users.sort((a, b) => a.age - b.age);       // youngest first
users.sort((a, b) => a.name.localeCompare(b.name)); // alphabetical

// ── FLATTENING NESTED ARRAYS ──
[1, [2, 3], [4, [5, 6]]].flat();     // [1, 2, 3, 4, [5, 6]] (1 level)
[1, [2, [3, [4]]]].flat(Infinity);  // [1, 2, 3, 4] (all levels)

map, filter, reduce, forEach
The most important array methods in modern JavaScript — master these completely

Why These Methods Matter So Much

These are functional, non-mutating, higher-order array methods. They take a callback function and apply it to every element. They are the backbone of modern JavaScript and React — you will use them constantly to transform data fetched from APIs into UI.

MethodReturnsUse when…
forEachundefinedYou just need to "do something" with each item (side effects)
mapNew array (same length)You want to transform each item into something new
filterNew array (≤ length)You want to keep only items matching a condition
reduceSingle valueYou want to combine all items into one result (sum, object, etc.)
find / findIndexSingle item / indexYou want the first match only
someBooleanYou want to check if AT LEAST ONE item matches
everyBooleanYou want to check if ALL items match
JavaScript — map, filter, reduce Complete
const products = [
  { id: 1, name: 'Laptop', price: 55000, category: 'electronics', inStock: true },
  { id: 2, name: 'Shirt',  price: 799,   category: 'clothing',   inStock: true },
  { id: 3, name: 'Phone',  price: 22000, category: 'electronics', inStock: false },
  { id: 4, name: 'Shoes',  price: 2499,  category: 'clothing',   inStock: true },
];

// ── forEach: loop with side effects, returns undefined ──
products.forEach(p => console.log(p.name));
// 'Laptop' 'Shirt' 'Phone' 'Shoes' — just logs, doesn't return anything

// ── map: TRANSFORM every item into something new ──
const names = products.map(p => p.name);
// ['Laptop', 'Shirt', 'Phone', 'Shoes']

const withTax = products.map(p => ({
  ...p,                          // spread existing properties
  priceWithTax: p.price * 1.18  // add new computed property
}));

// THE most common React pattern — render a list:
// products.map(p => <ProductCard key={p.id} product={p} />)

// ── filter: KEEP only items matching condition ──
const inStockOnly = products.filter(p => p.inStock);
// [Laptop, Shirt, Shoes] — Phone removed (inStock: false)

const electronics = products.filter(p => p.category === 'electronics');
const expensive   = products.filter(p => p.price > 5000);

// CHAINING filter + map (extremely common pattern)
const cheapElectronicNames = products
  .filter(p => p.category === 'electronics')
  .filter(p => p.price < 30000)
  .map(p => p.name);
// ['Phone']

// ── reduce: COMBINE all items into ONE value ──
// reduce(callback(accumulator, currentItem), initialValue)
const totalPrice = products.reduce((sum, p) => sum + p.price, 0);
// 55000+799+22000+2499 = 80298

const mostExpensive = products.reduce((max, p) =>
  p.price > max.price ? p : max
);
// the Laptop object (price: 55000)

// reduce to build an OBJECT — grouping by category
const byCategory = products.reduce((groups, p) => {
  const key = p.category;
  if (!groups[key]) groups[key] = [];
  groups[key].push(p);
  return groups;
}, {});
// { electronics: [Laptop, Phone], clothing: [Shirt, Shoes] }

// reduce to count occurrences
const counts = products.reduce((acc, p) => {
  acc[p.category] = (acc[p.category] || 0) + 1;
  return acc;
}, {});
// { electronics: 2, clothing: 2 }

// ── find / some / every ──
const laptop = products.find(p => p.name === 'Laptop');  // the object, or undefined

const hasOutOfStock = products.some(p => !p.inStock);  // true (Phone is out)
const allInStock     = products.every(p => p.inStock);  // false (Phone breaks it)

// ── CHAINING EVERYTHING TOGETHER (real-world pattern) ──
const summary = products
  .filter(p => p.inStock)
  .map(p => p.price)
  .reduce((sum, price) => sum + price, 0);
// Total value of in-stock products: 58298

Key-value pairs — how JavaScript models real-world entities
JavaScript — Objects Complete

  address: { city: 'Junagadh', state: 'Gujarat' },  // nested object

  }
};

// ── ACCESSING PROPERTIES ──
user.name              // dot notation — 'Poorv'
user['name']          // bracket notation — 'Poorv'
const key = 'age';
user[key]               // 21 — bracket notation needed for DYNAMIC keys
user.address.city       // 'Junagadh' — chaining for nested access
user.greet()            // "Hi, I'm Poorv"

// ── ADDING / UPDATING / DELETING ──
user.email = 'poorv@email.com';  // add new property
user.age = 22;                  // update existing
delete user.isStudent;            // remove property

// ── CHECKING IF PROPERTY EXISTS ──
'name' in user;              // true
user.hasOwnProperty('age'); // true
user.phone === undefined;     // true (doesn't exist)

// ── GETTING KEYS, VALUES, ENTRIES ──
Object.keys(user);       // ['name', 'age', 'address', 'skills', 'greet']
Object.values(user);     // ['Poorv', 22, {...}, [...], function]
Object.entries(user);     // [['name','Poorv'], ['age',22], ...]

// Iterate over an object's key-value pairs
for (const [key, value] of Object.entries(user)) {
  console.log(`${key}: ${value}`);
}

// ── MERGING OBJECTS ──
const defaults = { theme: 'light', fontSize: 14 };
const userPrefs = { fontSize: 18 };
const merged = { ...defaults, ...userPrefs };
// { theme: 'light', fontSize: 18 } — later spread overrides earlier
Object.assign({}, defaults, userPrefs); // same result, older syntax

// ── COPYING OBJECTS (be careful — shallow vs deep) ──
const shallowCopy = { ...user };    // shallow copy — nested objects still shared!
shallowCopy.address.city = 'Surat';  // ⚠️ this ALSO changes user.address.city!

const deepCopy = JSON.parse(JSON.stringify(user)); // simple deep copy (loses functions/dates)
const deepCopy2 = structuredClone(user); // modern, correct deep copy (no functions)

// ── FREEZING (immutability) ──
const config = Object.freeze({ apiUrl: 'https://api.com' });
config.apiUrl = 'hacked'; // silently fails (or throws in strict mode)

// ── COMPUTED PROPERTY NAMES (ES6) ──
const dynamicKey = 'role';
const obj = {
  [dynamicKey]: 'admin',      // key is the VALUE of dynamicKey variable
  [`is${dynamicKey}`]: true   // "isrole": true
};
// { role: 'admin', isrole: true }

// ── OPTIONAL CHAINING + NULLISH (covered Day 4, used heavily with objects) ──
const city = user.address?.city ?? 'Unknown';

Destructuring
Unpacking values from arrays and objects — used constantly in React

Why Destructuring Matters

Destructuring lets you extract multiple values from arrays or objects in a single statement. It dramatically reduces boilerplate code, and is the standard way props and state are accessed in React.

JavaScript — Destructuring Complete
// ── ARRAY DESTRUCTURING ──
const coords = [12.9, 77.6];
const [lat, lng] = coords;     // lat=12.9, lng=77.6

// Skipping items with commas
const [first, , third] = [1, 2, 3]; // first=1, third=3 (skip index 1)

// Swapping variables (classic trick)
let a = 1, b = 2;
[a, b] = [b, a];   // a=2, b=1 — no temp variable needed!

// Default values
const [x = 10, y = 20] = [5]; // x=5, y=20 (default used since no 2nd item)

// With rest pattern
const [winner, runnerUp, ...others] = ['Amy', 'Bob', 'Cat', 'Dan'];
// winner='Amy', runnerUp='Bob', others=['Cat','Dan']

// React's useState ALWAYS uses array destructuring:
// const [count, setCount] = useState(0);

// ── OBJECT DESTRUCTURING ──
const user = { name: 'Poorv', age: 21, city: 'Junagadh', country: 'India' };

const { name, age } = user;     // extract by KEY NAME (order doesn't matter)

// Renaming while destructuring
const { name: fullName, city: hometown } = user;
// fullName='Poorv', hometown='Junagadh'

// Default values
const { name: n, phone = 'Not provided' } = user;
// phone='Not provided' since user.phone doesn't exist

// Nested destructuring
const response = { data: { user: { name: 'Poorv', id: 42 } } };
const { data: { user: { name: userName, id } } } = response;
// userName='Poorv', id=42

// Rest pattern with objects
const { name: theName, ...rest } = user;
// theName='Poorv', rest={age:21, city:'Junagadh', country:'India'}

// ── DESTRUCTURING IN FUNCTION PARAMETERS (VERY common in React) ──
function greet({ name, age }) {     // destructure right in the parameter
  return `${name} is ${age} years old`;
}
greet(user); // "Poorv is 21 years old"

// React component props (THIS is the #1 use case):
// function ProductCard({ title, price, image }) {
//   return <div>{title} - ₹{price}</div>
// }

// Destructuring with default values in parameters
function createUser({ name, role = 'user', isActive = true } = {}) {
  return { name, role, isActive };
}
createUser({ name: 'Poorv' }); // { name:'Poorv', role:'user', isActive:true }

Spread & Rest Operators
The same ... syntax, two opposite jobs

Spread vs Rest — Same Symbol, Opposite Direction

Spread (...) EXPANDS an array/object into individual elements — used when calling functions or building new arrays/objects.
Rest (...) COLLECTS multiple elements INTO an array — used in function parameters or destructuring.

Quick rule: If ... is on the LEFT side of an assignment or in a function parameter, it's REST (collecting). If it's used to EXPAND something into a new array/object/argument list, it's SPREAD.
JavaScript — Spread & Rest in Practice
// ── SPREAD with arrays ──
const arr1 = [1, 2, 3];
const arr2 = [4, 5];
const combined = [...arr1, ...arr2];      // [1,2,3,4,5]
const copy = [...arr1];                 // shallow copy
const withExtra = [0, ...arr1, 4];     // [0,1,2,3,4]
Math.max(...arr1);                    // 3 — spread as function args

// ── SPREAD with objects ──
const base = { a: 1, b: 2 };
const extended = { ...base, c: 3 };       // {a:1, b:2, c:3}
const overridden = { ...base, a: 99 };    // {a:99, b:2} — later key wins

// Common React pattern: updating state immutably
// setUser(prevUser => ({ ...prevUser, name: 'New Name' }));

// ── SPREAD with strings ──
const chars = [...'hello'];   // ['h','e','l','l','o']

// ── REST in function parameters ──
function sum(...numbers) {       // collects ALL args into an array
  return numbers.reduce((a,b) => a+b, 0);
}
sum(1,2,3,4); // 10

// Combining named params + rest
function logScores(name, ...scores) {
  console.log(name, scores);
}
logScores('Poorv', 90, 85, 95); // 'Poorv' [90, 85, 95]

// ── REST in destructuring ──
const [first, ...rest] = [1, 2, 3, 4]; // first=1, rest=[2,3,4]
const { id, ...otherProps } = { id: 1, name: 'X', age: 5 };
// id=1, otherProps={name:'X', age:5}

// ── REAL PATTERN: removing a property immutably ──
const user = { id: 1, name: 'Poorv', password: 'secret123' };
const { password, ...safeUser } = user;
// safeUser = { id:1, name:'Poorv' } — password excluded, safe to send to client

ES6+ Modern Features
Template literals, classes, modules, Map/Set — the modern JS toolkit
JavaScript — ES6 Classes
// ── CLASSES (ES6) — syntactic sugar over prototypes ──
class Animal {
  constructor(name, sound) {
    this.name = name;
    this.sound = sound;
  }

  makeSound() {                      // method (no 'function' keyword needed)
    return `${this.name} says ${this.sound}`;
  }

  static create(name) {            // static method — called on class itself
    return new Animal(name, '...');
  }

  get info() {                       // getter — accessed like a property
    return `Animal: ${this.name}`;
  }
}

const dog = new Animal('Dog', 'Woof');
dog.makeSound();    // 'Dog says Woof'
dog.info;            // 'Animal: Dog' (getter — no parens!)
Animal.create('Cat'); // static method call

// ── INHERITANCE ──
class Dog extends Animal {
  constructor(name, breed) {
    super(name, 'Woof');     // call parent constructor
    this.breed = breed;
  }

  makeSound() {                  // override parent method
    return super.makeSound() + ' (loudly)'; // call parent's version too
  }
}
const labrador = new Dog('Rex', 'Labrador');
labrador.makeSound(); // 'Rex says Woof (loudly)'
labrador instanceof Animal; // true

// ── PRIVATE FIELDS (ES2022) ──
class BankAccount {
  #balance = 0;  // # makes it truly private — not accessible outside

  deposit(amount) { this.#balance += amount; }
  getBalance() { return this.#balance; }
}
const acc = new BankAccount();
acc.deposit(100);
acc.getBalance(); // 100
// acc.#balance; // ❌ SyntaxError — private, inaccessible from outside
JavaScript — Modules, Map, Set, Template Literals
// ── ES6 MODULES (import/export) ──
// In math.js:
export const PI = 3.14159;
export function add(a, b) { return a + b; }
export default function multiply(a, b) { return a * b; } // only ONE default export per file

// In main.js:
import multiply, { PI, add } from './math.js';  // default + named imports
import * as mathUtils from './math.js';     // import everything as namespace

// ── MAP — like objects, but keys can be ANY type, preserves insertion order ──
const userRoles = new Map();
userRoles.set('poorv', 'admin');
userRoles.set('riya', 'editor');
userRoles.get('poorv');     // 'admin'
userRoles.has('riya');     // true
userRoles.delete('riya');  // removes entry
userRoles.size;             // number of entries
for (const [key, val] of userRoles) { console.log(key, val); }

// ── SET — collection of UNIQUE values ──
const uniqueIds = new Set([1, 2, 2, 3, 3, 3]);
console.log(uniqueIds);  // Set {1, 2, 3} — duplicates auto-removed
uniqueIds.add(4);
uniqueIds.has(2);       // true
[...uniqueIds];        // [1, 2, 3, 4] — convert back to array

// Common pattern: remove duplicates from an array
const nums = [1, 2, 2, 3, 1, 4];
const unique = [...new Set(nums)];  // [1, 2, 3, 4]

// ── TEMPLATE LITERALS — tagged templates, multiline ──

const card = `
  <div class="card">
    <h3>${name}</h3>
    <p>${role}</p>
  </div>
`;  // multi-line string with embedded expressions — common for HTML generation

JSON — JavaScript Object Notation
The universal data format for APIs — every backend speaks this

Why JSON Matters for Full Stack

JSON is a lightweight, text-based data format derived from JavaScript object syntax — but it's language-independent. Every REST API you'll build or consume sends and receives JSON. Understanding JSON.stringify and JSON.parse is essential for all API work.

JavaScript — JSON Complete
// ── JSON RULES ──
// - Keys MUST be double-quoted strings
// - Values can be: string, number, boolean, null, object, array
// - NO functions, NO undefined, NO comments, NO trailing commas

const jsonString = '{"name":"Poorv","age":21,"skills":["JS","React"]}';

// ── JSON.parse: STRING → JavaScript OBJECT ──
const obj = JSON.parse(jsonString);
obj.name;    // 'Poorv' — now a real JS object you can use

// ── JSON.stringify: JavaScript OBJECT → STRING ──
const data = { name: 'Poorv', age: 21, isActive: true };
const str = JSON.stringify(data);
// '{"name":"Poorv","age":21,"isActive":true}'

// Pretty-print with indentation (useful for debugging/logging)
JSON.stringify(data, null, 2);
/* {
  "name": "Poorv",
  "age": 21,
  "isActive": true
} */

// ── REAL-WORLD USE: Fetching from an API ──
async function getUser() {
  const response = await fetch('https://api.example.com/user/1');
  const user = await response.json();  // parses JSON response automatically
  console.log(user.name);
}

// ── SENDING JSON in a POST request ──
async function createUser(userData) {
  const response = await fetch('https://api.example.com/users', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(userData)  // MUST stringify before sending
  });
  return response.json();
}

// ── localStorage works only with strings — use JSON to store objects ──
const cart = [{ id: 1, name: 'Laptop' }];

const savedCart = JSON.parse(localStorage.getItem('cart') || '[]');

The 'this' Keyword
The most confusing concept in JavaScript — finally explained clearly

The Core Rule

Unlike most languages, this in JavaScript is determined by how a function is called, not where it's defined (except for arrow functions). This is called "dynamic this binding."

JavaScript — 'this' in 5 Contexts
// ── 1. GLOBAL CONTEXT ──
console.log(this); // in browser: Window object (in strict mode/modules: undefined)

// ── 2. INSIDE A REGULAR METHOD — 'this' = the object before the dot ──
const user = {
  name: 'Poorv',
  greet() {
    console.log(this.name); // 'this' = user object → 'Poorv'
  }
};
user.greet(); // 'Poorv'

// ── 3. LOSING CONTEXT (a VERY common bug!) ──
const greetFn = user.greet;
greetFn(); // ❌ undefined — 'this' is now undefined/window, NOT user!
// Why? Because greetFn is called standalone, with no object before the dot.

// ── 4. ARROW FUNCTIONS — 'this' is INHERITED from enclosing scope ──
const user2 = {
  name: 'Poorv',
  greet: () => {
    console.log(this.name); // ❌ undefined! Arrow fn took 'this' from OUTSIDE (global)
  }
};
// Arrow functions don't have their OWN 'this' — they use the surrounding scope's.
// This is why arrow functions should NOT be used as object methods.

// BUT arrow functions are PERFECT inside regular methods (common React/event pattern):
class Counter {
  count = 0;
  increment = () => {        // arrow function as class property
    this.count++;            // 'this' correctly refers to the Counter instance
  }
}
// Without arrow function, 'this' inside a callback (like setTimeout, onClick)
// often loses its binding — arrow functions fix this by inheriting 'this'.

// ── 5. EXPLICIT BINDING: call, apply, bind ──
function introduce() {
  return `I am ${this.name}`;
}
const person1 = { name: 'Poorv' };
const person2 = { name: 'Riya' };

introduce.call(person1);          // 'I am Poorv' — call(thisArg, ...args)
introduce.apply(person2);         // 'I am Riya' — apply(thisArg, [argsArray])
const boundFn = introduce.bind(person1); // bind RETURNS a new function with 'this' locked
boundFn();                       // 'I am Poorv' — can call anytime, 'this' is fixed

// ── REAL-WORLD FIX for losing 'this' in event handlers ──
class Button {
  constructor() {
    this.label = 'Click me';
    // Fix 1: bind in constructor
    this.handleClick = this.handleClick.bind(this);
  }
  handleClick() {
    console.log(this.label); // correctly bound now
  }
}
// Fix 2 (modern, simpler): use arrow function as class property
// handleClick = () => { console.log(this.label); }
Interview Trap: "Why does this lose its value when I pass a method as a callback?" — Answer: because this depends on HOW a function is called (the calling context), not where it's defined. Passing obj.method as a callback strips it from obj — it becomes a standalone function call. Fix with arrow functions, .bind(), or by calling it as () => obj.method().

Real Project: Shopping Cart Logic
Combining everything learned today into one realistic feature
🏢 Real World

This is exactly what e-commerce companies build

Shopping cart logic — adding items, removing items, calculating totals, applying discounts — uses arrays, objects, destructuring, spread, and array methods together. This is a common take-home assignment or live coding question.

JavaScript — Shopping Cart (Real Project Pattern)
class ShoppingCart {
  #items = [];  // private field — array of { product, quantity }

  addItem(product, quantity = 1) {
    const existing = this.#items.find(item => item.product.id === product.id);

    if (existing) {
      existing.quantity += quantity; // already in cart — increase quantity
    } else {
      this.#items.push({ product, quantity }); // new item
    }
    return this; // enable method chaining
  }

  removeItem(productId) {
    this.#items = this.#items.filter(item => item.product.id !== productId);
    return this;
  }

  updateQuantity(productId, quantity) {
    const item = this.#items.find(i => i.product.id === productId);
    if (item) item.quantity = Math.max(0, quantity);
    return this;
  }

  get itemCount() {
    return this.#items.reduce((sum, item) => sum + item.quantity, 0);
  }

  get subtotal() {
    return this.#items.reduce(
      (sum, item) => sum + (item.product.price * item.quantity), 0
    );
  }

  applyDiscount(percent) {
    return this.subtotal * (1 - percent / 100);
  }

  getSummary() {
    return this.#items.map(({ product, quantity }) => ({
      name: product.name,
      quantity,
      lineTotal: product.price * quantity
    }));
  }
}

// ── USAGE ──
const cart = new ShoppingCart();

cart
  .addItem({ id: 1, name: 'Laptop', price: 55000 }, 1)
  .addItem({ id: 2, name: 'Mouse',  price: 499 },  2)
  .addItem({ id: 1, name: 'Laptop', price: 55000 }, 1); // adds to existing

console.log(cart.itemCount);   // 4 (2 laptops + 2 mice)
console.log(cart.subtotal);    // 110998
console.log(cart.applyDiscount(10)); // 99898.2 (10% off)
console.log(cart.getSummary());
// [{name:'Laptop', quantity:2, lineTotal:110000}, {name:'Mouse', quantity:2, lineTotal:998}]

Placement Interview Q&A
Top intermediate JavaScript questions asked in Ahmedabad company interviews
  • Q1. What is the difference between map and forEach?
    map returns a NEW array containing the result of calling the callback on every element — it's used for transformation. forEach returns undefined — it's used purely for side effects (like logging or making API calls per item) and doesn't build a new array. If you need to use the result, use map. If you're just performing an action without needing a returned array, use forEach. A related point: map (and filter, reduce) are chainable; forEach is not since it returns undefined.
  • Q2. Explain reduce() with a real example.
    reduce() takes a callback (accumulator, currentValue) and an initial value, and "reduces" an entire array down to a single value by repeatedly applying the callback. For example, to sum an array: arr.reduce((sum, n) => sum + n, 0). The accumulator carries forward the running result through each iteration. It's also used to build objects (grouping data by category), find min/max, or flatten arrays — anywhere you need to combine many values into one.
  • Q3. What is destructuring and why is it useful?
    Destructuring is a syntax for unpacking values from arrays or properties from objects into individual variables in a single statement. For arrays: const [a, b] = [1, 2]. For objects: const { name, age } = user. It reduces boilerplate (no need to write user.name, user.age repeatedly), supports default values and renaming, and is the standard way React components receive props: function Card({ title, price }) {}.
  • Q4. What's the difference between spread and rest operators? They look the same.
    Both use the ... syntax but do opposite things depending on context. Spread EXPANDS an iterable into individual elements — used when building a new array/object or passing arguments: [...arr1, ...arr2] or Math.max(...numbers). Rest COLLECTS multiple arguments or remaining destructured items INTO a single array — used in function parameters or destructuring: function sum(...nums) or const [first, ...rest] = arr. The simple way to tell them apart: rest appears where you're "receiving" values (parameters, destructuring left-side); spread appears where you're "giving" or expanding values.
  • Q5. How do you remove duplicates from an array?
    The cleanest modern way is using a Set, which only stores unique values: [...new Set(array)]. This converts the array to a Set (auto-removing duplicates), then spreads it back into a new array. For arrays of primitive values (numbers, strings) this works perfectly. For arrays of objects, you'd need a different approach since objects are compared by reference — typically using a Map keyed by a unique property, or filter with findIndex to check first occurrence.
  • Q6. What is the difference between shallow copy and deep copy?
    A shallow copy (using spread {...obj} or Object.assign) copies only the top-level properties. If a property's value is itself an object or array, both the original and copy share a reference to that SAME nested object — mutating it through one affects the other. A deep copy creates entirely independent copies of nested structures too, so mutations don't leak across. Deep copies can be made with structuredClone(obj) (modern, recommended) or JSON.parse(JSON.stringify(obj)) (older, loses functions and special types like Date).
  • Q7. Why does 'this' sometimes become undefined inside a method?
    'this' in JavaScript is determined by HOW a function is called, not where it's defined. If you call obj.method(), 'this' is obj. But if you extract the method (const fn = obj.method) and call it standalone (fn()), 'this' is lost — it becomes undefined (in strict mode) because there's no object before the dot at call time. This commonly happens when passing a method as a callback (e.g., to setTimeout or an event listener). Fixes: use .bind(obj), wrap in an arrow function () => obj.method(), or define the method as an arrow function class property which inherits 'this' from the surrounding scope.
  • Q8. What is JSON and why is it used in APIs?
    JSON (JavaScript Object Notation) is a lightweight, text-based data interchange format. Despite the name, it's language-independent — virtually every programming language can parse and generate it. APIs use JSON because it's human-readable, maps cleanly to common data structures (objects, arrays, strings, numbers, booleans, null), and is much lighter than older formats like XML. In JavaScript, JSON.stringify() converts an object to a JSON string (for sending in a request body), and JSON.parse() converts a JSON string back into a usable JavaScript object (for reading a response).
